# 介護記録アプリ (Spring Boot) 本番環境デプロイ作業指示書

**構成**: アプリ実行環境（Fly.io） + データベース（Neon PostgreSQL）
**方針**: ブラックボックスを排除し、CLIからすべての動作・ログを把握できる確実な手順とする。

---

## ステップ1：Neon（PostgreSQL）の構築と接続情報の取得

※ この工程のみGUI（ブラウザ）で実施。

1. **プロジェクト作成とマイグレーション**
   - [x] [Neon公式サイト](https://neon.tech/) にログイン。 カウントがなければ作成
   - [x] プロジェクト名（例：`care-record-db`）、Postgresバージョン（16）、リージョン（`Asia Pacific (Singapore)`等）を指定して作成。
   - [x] Neon管理画面の「SQL Editor」で以下を実行する。
      - [x] DDL: `docker/postgres/init/01_init_schema.sql` 
      - [x] マスター: `master追加.sql`
2. **接続情報（.env形式）の取得**
   - [x] ダッシュボードの「Connect」モーダル等から、以下の5つの値を確認・メモしておく。
      Neonの公式ドキュメントConnect from any applicationも確認(https://neon.com/docs/connect/connect-from-any-app)
     - `PGHOST` (例: ep-cool-...-pooler.us-east-2.aws.neon.tech)
     - `PGDATABASE`
     - `PGUSER`
     - `PGPASSWORD`
     - `PGPORT` (通常は 5432)

## ステップ2：Spring Boot側の準備（properties設定 ＆ Dockerfile作成）

Neonの環境変数を使って、Spring Boot側で自動的にJDBC URLを構築する設計にする。

### 1. `src/main/resources/application.properties` の設定変更
- [x] 本番用の環境変数をバインドするため、ファイル内を以下のように記述する。

```properties
# --- データベース接続設定（Neonの環境変数を結合してJDBC URLを動的生成） ---
spring.datasource.url=jdbc:postgresql://${PGHOST}:${PGPORT}/${PGDATABASE}?sslmode=require
spring.datasource.username=${PGUSER}
spring.datasource.password=${PGPASSWORD}
# アイドル状態の接続を0にする
spring.datasource.hikari.minimum-idle=0
# 接続のアイドルタイムアウトを短縮（例: 2分、Neonの5分スリープより前に閉じる）
spring.datasource.hikari.idle-timeout=120000
# タイムアウトの設定
spring.datasource.hikari.connection-timeout=30000

# --- LINE Bot 設定（Fly.ioの環境変数から取得） ---
line.bot.channel-token=${LINE_CHANNEL_TOKEN}
line.bot.channel-secret=${LINE_CHANNEL_SECRET}
line.bot.handler.path=/api/v1/webhook/line
```

### 2. Dockerfile の作成
- [x] Spring Bootのプロジェクトルート（`build.gradle` がある階層）に以下の `Dockerfile` を作成する。

```dockerfile
# 1. 実行環境には軽量なJREを指定（Java 21）
FROM eclipse-temurin:21-jre-alpine

# 2. 作業ディレクトリの指定
WORKDIR /app

# 3. 事前にビルドしたjarファイルをコピー
# ファイル名を直接指定する（一番確実、実際のファイル名は要確認）
COPY build/libs/care-record-bot-0.0.1-SNAPSHOT.jar app.jar

# 4. コンテナ起動時のコマンド
ENTRYPOINT ["java", "-jar", "app.jar"]

# Fly.io にポートを知らせる
EXPOSE 8080
```

## ステップ3：Fly.io CLIのインストールとアプリ登録

以降はプロジェクトルートのターミナルで実行する。

- [x] 1. **Fly.io CLI（`flyctl`）のインストール**
   - Homebrewを使用してインストールします。Fly.ioアカウントなければ作成。githubが楽
   ```bash
   brew install flyctl
   ```
   - [x] インストール後、コマンドが通るか確認します。バージョン情報が出れば成功です。
   ```bash
   fly version
   ```
- [x] 2. **ログイン**
   ```bash
   fly auth login
   ```
   - ブラウザが開くので、GitHubアカウント等でログインを完了させます。
- [x] 3. **アプリの初期化（※まだデプロイしない）**
   ```bash
   fly launch --no-deploy
   ```
   - [x] アプリ名を入力（例：`care-record-bot`）。
     - **【重要: Fly.io特有のルール】** アプリ名は世界中のユーザー間で**完全一意（早い者勝ち）**である必要があります。
     - もし `Error App name is already taken` と出た場合は、`care-record-bot-xxxx` のようにサフィックス（任意の文字列）を付けて再度入力してください。
   - [x] リージョンを選択（例：`Tokyo, Japan (nrt)`）。
   - [x] 実行後、カレントディレクトリに `fly.toml` が生成されることを確認する。

## ステップ4：環境変数（Secret）の注入

- [x] ローカルの `.env` に代わる機密情報を、Fly.ioの暗号化領域（Secrets）に直接登録する。
- [x] ここでセットした値が、ステップ2の `${PGHOST}` 等に自動的に注入される。

```bash
# DBの接続情報（ステップ1でメモしたNeonの情報をそのまま登録）
fly secrets set PGHOST="[Neonのホスト名]"
fly secrets set PGDATABASE="[データベース名]"
fly secrets set PGUSER="[ユーザー名]"
fly secrets set PGPASSWORD="[パスワード]"
fly secrets set PGPORT="5432"

# LINEの認証情報（application.propertiesの変数名と完全一致させる）
fly secrets set LINE_CHANNEL_TOKEN="LINEのアクセストークン"
fly secrets set LINE_CHANNEL_SECRET="LINEのシークレット"
```

## ステップ5：ビルドと本番デプロイ

ローカルで確実なjarを生成し、それをFly.ioへ送り込む。

- [x] 1. **jarファイルのビルド**
   ```bash
   ./gradlew clean bootJar
   ```
- [x] 2. **デプロイの実行**
   ```bash
   fly deploy
   ```
   - ターミナル上のビルド・デプロイログを監視し、エラーなく完了するか確認する。

## ステップ6：動作確認とログ監視

デプロイ完了後、本番環境の生ログをターミナルで監視し、正常起動を確認する。

```bash
fly logs
```
- [x] Spring Bootの起動バナー、HikariCPの接続ログ等が流れ、エラーで停止していないことを確認。
- [ ] 正常起動が確認できたら、Fly.ioから割り当てられたURL（例：`https://[アプリ名].fly.dev/api/v1/webhook/line`）を、LINE公式アカウントのWebhook URLに設定する。
