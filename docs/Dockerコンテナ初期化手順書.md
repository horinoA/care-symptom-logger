## ✅ 正しい手順（完全版）
正解は、「最初の docker-compose up -d を叩く前に、絶対にSQLファイルを所定の場所に配置しておくこと」です。

1. docker-compose.yml をルートディレクトリに保存する。
手作業で ルートに docker/postgres/init というフォルダを作成する。（※Dockerが勝手に作るのを待たずに自分で作ります）
そのフォルダの中に 01_init_schema.sql を保存する。

2. ここで初めて docker-compose up -d を叩く！
この順番でやれば、PostgreSQLが「初回起動だ！お、SQLファイルもあるぞ！全部実行してから起動しよう！」と完璧に動いてくれます。

### 💣 もしすでに手順「2」をやってしまった場合のリカバリー
もしすでに空っぽで up してしまった場合は、初期化フラグをへし折るために、以下のコマンドでデータを物理削除してから「正しい手順の4」をやり直してください。
```Bash
# コンテナを停止
docker-compose down

# 自動生成された初期化済みデータを完全に消し飛ばす
rm -rf db_data/*

# SQLファイルを配置してあることを確認してから、再度起動！
docker-compose up -d
```
